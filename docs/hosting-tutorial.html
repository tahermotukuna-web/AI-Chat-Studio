Overview

AI Chat Studio v2.0.0 is a lightweight, client-side application that can be hosted on various platforms. This tutorial covers multiple deployment scenarios from local development to production hosting.

Key Point: AI Chat Studio v2.0.0 is a static web application with PHP backend for API proxying. No database or complex server setup required!

System Requirements

Minimum Requirements

  • Web Server: Apache 2.4+ or Nginx 1.18+
  • PHP: PHP 7.4+ (PHP 8.0+ recommended)
  • Extensions: cURL, JSON, mbstring
  • Disk Space: 50MB minimum
  • Memory: 128MB PHP memory limit

Recommended Requirements

  • Web Server: Apache 2.4+ or Nginx 1.20+
  • PHP: PHP 8.0+
  • Extensions: cURL, JSON, mbstring, OpenSSL
  • Disk Space: 100MB+
  • Memory: 256MB PHP memory limit
  • SSL Certificate: For secure API communication

Local Development

Option 1: PHP Built-in Server (Recommended for Development)

Perfect for local development and testing:

# Navigate to the v2.0.0 directory
cd v2.0.0

# Start the PHP development server
php -S localhost:8001

# Access at http://localhost:8001
Done! Your local development server is running. Open your browser and go to http://localhost:8001

Option 2: XAMPP/WAMP/MAMP

For Windows/Mac users who prefer a full stack solution:

  1. Download and install XAMPP (or WAMP/MAMP)
  2. Copy the v2.0.0 folder to htdocs directory
  3. Start Apache from XAMPP control panel
  4. Access at http://localhost/v2.0.0

Option 3: Using Composer (Advanced)

If you want to add dependencies or customize the build:

# Install dependencies (if any)
composer install

# Start development server
php -S localhost:8001 -t v2.0.0

Deployment Steps

Step 1: Prepare Files

  1. Download the v2.0.0 folder from the repository
  2. Zip the entire v2.0.0 folder
  3. Ensure all files are included, especially the api/ directory

Step 2: Upload to Hosting

  1. Log into your hosting control panel (cPanel/Plesk)
  2. Open File Manager or connect via FTP/SFTP
  3. Navigate to public_html or www directory
  4. Upload and extract the zip file
  5. Ensure the directory structure is correct:
    public_html/
    ├── index.php
    ├── api/
    ├── assets/
    ├── pages/
    └── vendor/ (if needed)

Step 3: Configure Permissions

Set correct file permissions:

# Files should be 644
find /path/to/v2.0.0 -type f -exec chmod 644 {} \;

# Directories should be 755
find /path/to/v2.0.0 -type d -exec chmod 755 {} \;

Step 4: Test Installation

  1. Visit your domain in a web browser
  2. You should see the AI Chat Studio interface
  3. Test with a simple API key to ensure PHP backend works
Important: Make sure your shared hosting provider supports PHP 7.4+ and has the required extensions (cURL, JSON, mbstring).

VPS/Dedicated Server

For better performance and control, deploy on a VPS or dedicated server.

Ubuntu/Debian Setup

Step 1: Update System

sudo apt update && sudo apt upgrade -y

Step 2: Install LAMP Stack

# Install Apache
sudo apt install apache2 -y

# Install MySQL (optional, not required for AI Chat Studio)
# sudo apt install mysql-server -y

# Install PHP and extensions
sudo apt install php libapache2-mod-php php-curl php-json php-mbstring php-xml -y

# Enable Apache modules
sudo a2enmod rewrite
sudo systemctl restart apache2

Step 3: Deploy Application

# Download application
cd /var/www/html
sudo wget [your-download-link] -O ai-chat-studio.zip
sudo unzip ai-chat-studio.zip
sudo mv v2.0.0/* .
sudo rm -rf v2.0.0 ai-chat-studio.zip

# Set permissions
sudo chown -R www-data:www-data /var/www/html
sudo chmod -R 755 /var/www/html

Nginx Setup (Alternative)

# Install Nginx
sudo apt install nginx -y

# Install PHP-FPM
sudo apt install php-fpm php-curl php-json php-mbstring -y

# Configure Nginx (edit /etc/nginx/sites-available/default)
# Then restart Nginx
sudo systemctl restart nginx

Docker Deployment

For containerized deployment, create a Dockerfile:

Dockerfile

FROM php:8.0-apache

# Install PHP extensions
RUN docker-php-ext-install pdo_mysql mbstring exif pcntl bcmath gd

# Enable Apache modules
RUN a2enmod rewrite

# Copy application
COPY v2.0.0/ /var/www/html/

# Set permissions
RUN chown -R www-data:www-data /var/www/html
RUN chmod -R 755 /var/www/html

# Expose port
EXPOSE 80

CMD ["apache2-foreground"]

Docker Compose

version: '3.8'
services:
  web:
    build: .
    ports:
      - "80:80"
    volumes:
      - ./v2.0.0:/var/www/html
    environment:
      - APACHE_DOCUMENT_ROOT=/var/www/html
  nginx:
    image: nginx:alpine
    ports:
      - "443:443"
      - "80:80"
      volumes:
        - ./v2.0.0:/usr/share/nginx/html
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./ssl:/etc/nginx/ssl

SSL Certificate Setup

SSL is essential for secure API communication and user trust.

Free SSL with Let's Encrypt

Using Certbot (Ubuntu/Debian)

# Install Certbot
sudo apt install certbot python3-certbot-apache -y

# Get certificate
sudo certbot --apache -d yourdomain.com -d www.yourdomain.com

# Test auto-renewal
sudo certbot renew --dry-run

Using SSL Provider

  1. Purchase SSL certificate from provider
  2. Generate CSR (Certificate Signing Request)
  3. Submit CSR to certificate authority
  4. Download certificate files
  5. Configure web server with certificate

Apache SSL Configuration


    ServerName yourdomain.com
    DocumentRoot /var/www/html
    
    SSLEngine on
    SSLCertificateFile /path/to/certificate.crt
    SSLCertificateKeyFile /path/to/private.key
    SSLCertificateChainFile /path/to/chain.crt
    
    # Security headers
    Header always set Strict-Transport-Security "max-age=63072000; includeSubDomains; preload"
    Header always set X-Frame-Options DENY
    Header always set X-Content-Type-Options nosniff


# Redirect HTTP to HTTPS

    ServerName yourdomain.com
    Redirect permanent / https://yourdomain.com/
Important: Always use HTTPS in production to ensure API keys and conversations are transmitted securely.

Troubleshooting

Common Issues

Issue: "500 Internal Server Error"

Solution:
  • Check PHP error logs: /var/log/apache2/error.log
  • Verify PHP version: php -v
  • Check file permissions
  • Ensure all required PHP extensions are installed

Issue: API Calls Not Working

Solution:
  • Check if cURL is enabled: php -m | grep curl
  • Verify SSL certificates are working
  • Check network connectivity
  • Test API endpoints directly

Issue: "Page Not Found" on Subdirectories

Solution:
  • Ensure Apache mod_rewrite is enabled
  • Check .htaccess file exists and has correct permissions
  • Verify Apache AllowOverride is set to "All"

Issue: Slow Loading

Solution:
  • Enable PHP OPcache
  • Enable browser caching
  • Optimize images and assets
  • Use CDN for static assets

Performance Optimization

PHP Optimization

# In php.ini
opcache.enable=1
opcache.memory_consumption=128
opcache.max_accelerated_files=4000
opcache.revalidate_freq=2

# PHP-FPM optimization (www.conf)
pm=dynamic
pm.max_children=10
pm.start_servers=2
pm.min_spare_servers=1
pm.max_spare_servers=3

Apache Optimization

# Enable compression
LoadModule deflate_module modules/mod_deflate.so

    SetOutputFilter DEFLATE
    SetEnvIfNoCase Request_URI \.(?:gif|jpe?g|png)$ no-gzip dont-vary
    SetEnvIfNoCase Request_URI \.(?:exe|t?gz|zip|bz2|sit|rar)$ no-gzip dont-vary


# Enable browser caching
LoadModule expires_module modules/mod_expires.so
ExpiresActive On
ExpiresByType text/css "access plus 1 month"
ExpiresByType application/javascript "access plus 1 month"
ExpiresByType image/png "access plus 1 month"
ExpiresByType image/jpg "access plus 1 month"
ExpiresByType image/jpeg "access plus 1 month"

Backup Strategy

# Create backup script
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/backup"
APP_DIR="/var/www/html"

# Create backup
tar -czf $BACKUP_DIR/ai-chat-studio-$DATE.tar.gz $APP_DIR

# Keep only last 30 days
find $BACKUP_DIR -name "ai-chat-studio-*.tar.gz" -mtime +30 -delete

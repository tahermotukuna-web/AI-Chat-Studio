Get AI Chat Studio up and running in 5 minutes

Overview

AI Chat Studio v2.0.0 is a lightweight, client-side application that provides a clean interface for AI chat interactions. This quick start guide will get you running in just a few minutes.

Time Required: 5-10 minutes | Difficulty: Beginner | Prerequisites: Basic computer skills

What You'll Accomplish

  • Download and set up AI Chat Studio v2.0.0
  • Configure your first AI provider
  • Send your first chat message
  • Understand basic configuration options

Prerequisites

System Requirements

Local Testing

  • Operating System: Windows, macOS, or Linux
  • PHP: Version 7.4+ (8.0+ recommended)
  • Web Server: PHP built-in server or Apache
  • Browser: Modern web browser (Chrome, Firefox, Safari, Edge)

Production Hosting

  • Web Server: Apache 2.4+ or Nginx 1.18+
  • PHP: Version 7.4+ with cURL extension
  • SSL Certificate: Recommended for security
  • Storage: 50MB minimum disk space

AI Provider Accounts

You'll need an API key from at least one AI provider:

  • OpenAI: GPT-3.5/GPT-4 models
  • Anthropic: Claude models
  • Google: Gemini models
  • Local Models: Ollama, LM Studio
Security Note: Never share your API keys or commit them to version control. Use environment variables or secure configuration files.

Download & Extract

Step 1: Download the Application

Download AI Chat Studio v2.0.0 from the official repository:

# Using Git (recommended) git clone https://github.com/your-repo/ai-chat-studio.git cd ai-chat-studio # Or download and extract ZIP # Download from releases page and extract to your desired location

Step 2: Verify Download

Ensure you have the correct directory structure:

ai-chat-studio/ ├── v2.0.0/ # Latest version (recommended) │ ├── index.php │ ├── api/ │ ├── assets/ │ └── pages/ ├── main/ # Alternative version └── docs/ # Documentation
Recommended: Use the v2.0.0 directory for the latest features and improvements.

Basic Setup

1Start Local Server

Navigate to the v2.0.0 directory and start the PHP development server:

# Navigate to v2.0.0 directory cd v2.0.0 # Start development server php -S localhost:8001 # Server will start at http://localhost:8001

2Access the Application

Open your web browser and navigate to the local server:

  • Go to http://localhost:8001
  • You should see the AI Chat Studio interface
  • If you see an error, check the terminal for PHP error messages

3Configure API Provider

To start using AI chat features, you need to configure an API provider:

Option A: OpenAI Setup

// In the settings or configuration section: Provider: OpenAI API Key: sk-your-openai-api-key-here Model: gpt-3.5-turbo (or gpt-4)

Option B: Anthropic Claude Setup

// In the settings or configuration section: Provider: Anthropic API Key: sk-ant-your-anthropic-api-key-here Model: claude-3-sonnet-20240229
Important: API keys are sensitive information. Use environment variables in production and never hardcode them in your application files.

First Test

Send Your First Message

  1. Locate the Chat Interface: Find the text input field at the bottom of the page
  2. Enter a Test Message: Try something simple like "Hello, how are you?"
  3. Send the Message: Press Enter or click the Send button
  4. Wait for Response: The AI should respond within a few seconds

Verify Everything Works

Success Indicators:
  • Chat interface loads without errors
  • You can type and send messages
  • AI provider responds with relevant answers
  • Conversation history is maintained
  • No JavaScript or PHP errors in browser console

Common First-Use Issues

Issue: "Connection Error" or "Failed to fetch"

Solutions:
  • Check your API key is valid and has sufficient credits
  • Verify the API endpoint URL is correct
  • Ensure your internet connection is stable
  • Check browser console for detailed error messages

Issue: "PHP Error" or "Server Error"

Solutions:
  • Verify PHP is running: php -v
  • Check PHP extensions: php -m | grep curl
  • Ensure the server is running on the correct port
  • Check file permissions in the v2.0.0 directory

Issue: "CORS Error" or "Access Denied"

Solutions:
  • Access via http://localhost:8001 not file://
  • Check if your hosting provider supports the required PHP version
  • Verify API provider allows requests from your domain

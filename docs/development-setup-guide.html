Git Setup

Step 1: Fork and Clone Repository

Option A: Fork Method (Recommended for Contributions)

# 1. Fork the repository on GitHub # Visit: https://github.com/your-repo/ai-chat-studio # Click "Fork" button # 2. Clone your fork git clone https://github.com/your-username/ai-chat-studio.git cd ai-chat-studio # 3. Add upstream remote git remote add upstream https://github.com/original-repo/ai-chat-studio.git # 4. Verify remotes git remote -v

Option B: Direct Clone (For Personal Use)

# Clone directly git clone https://github.com/your-repo/ai-chat-studio.git cd ai-chat-studio

Step 2: Configure Git

# Set your identity git config --global user.name "Your Name" git config --global user.email "your.email@example.com" # Set default branch name git config --global init.defaultBranch main # Set pull behavior git config --global pull.rebase false # Set line endings (Windows) git config --global core.autocrlf true # Set line endings (Linux/macOS) git config --global core.autocrlf input

Step 3: Create Development Branch

# Create and switch to development branch git checkout -b development # Or for feature branches git checkout -b feature/your-feature-name # Set upstream for new branch git push -u origin development
Branch Strategy: Use main for stable releases, development for integration, and feature/feature-name for individual features.

Project Structure

Understanding the project structure is crucial for efficient development. AI Chat Studio follows a modular architecture:

ai-chat-studio/ ├── docs/ # Documentation │ ├── quick-start-guide.html │ ├── api-configuration-guide.html │ ├── development-setup-guide.html │ └── hosting-tutorial.html ├── v2.0.0/ # Latest version (main development) │ ├── api/ # Backend API endpoints │ │ └── chat.php # Main chat API handler │ ├── assets/ # Frontend assets │ │ ├── css/ # Stylesheets │ │ ├── js/ # JavaScript files │ │ ├── images/ # Image assets │ │ └── fonts/ # Font files │ ├── pages/ # Additional pages │ │ ├── about.php │ │ ├── settings.php │ │ └── faqs.php │ ├── index.php # Main entry point │ ├── manifest.json # PWA manifest │ └── vendor/ # Composer dependencies ├── main/ # Alternative version ├── v1.0.0/ # Legacy version ├── tests/ # Unit tests ├── composer.json # PHP dependencies ├── package.json # Node.js dependencies ├── .gitignore # Git ignore rules ├── .env.example # Environment template └── README.md # Project documentation

Key Files and Directories

Core Application Files

  • index.php - Main entry point, handles routing and initialization
  • api/chat.php - Core chat API endpoint, handles AI provider communication
  • assets/js/app.js - Main JavaScript application logic
  • assets/css/style.css - Main stylesheet

Configuration Files

  • .env.example - Environment variables template
  • composer.json - PHP dependencies and autoloading
  • package.json - Node.js dependencies for build tools
Important: Never commit sensitive data like API keys or real .env files. Use .env.example as a template.

Option A: PHP Built-in Server (Recommended)

PHP Coding Standards

PSR-12 Compliance

provider = $provider; $this->config = $config; } /** * Process chat request * * @param array $request * @return array * @throws Exception */ public function processRequest(array $request): array { try { $this->validateRequest($request); $response = $this->provider->chat($request['message']); return [ 'success' => true, 'data' => $response, 'timestamp' => time() ]; } catch (Exception $e) { return [ 'success' => false, 'error' => $e->getMessage(), 'timestamp' => time() ]; } } /** * Validate incoming request * * @param array $request * @return void * @throws Exception */ private function validateRequest(array $request): void { if (empty($request['message'])) { throw new Exception('Message is required'); } if (strlen($request['message']) > 4000) { throw new Exception('Message too long'); } } }

Documentation Standards

  • Use PHPDoc for all classes, methods, and functions
  • Include parameter types and return types
  • Document exceptions that can be thrown
  • Keep docblocks up to date with code changes
